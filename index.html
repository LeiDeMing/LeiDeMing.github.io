来自构造函数的对象

目前，我所说知道的两种创建对象的方法，对象字面量和new Object。其中，对象字面量方法更简单明了，另一个原因是它只是强调了该对象仅是一个可变哈希映射，而不是从对象中提取属性和方法。与new Object相比，对象字面量没有作用域解析，因为它可能创造了一个一样的局部构造函数，解析器需要从调用Object的位置一直向上查找，直到找到object的全局构造函数。

var car = { goes: "far"} // 对象字面量发

//构造函数法
var car = new Object();
car.goes = far;

Array.prototype.reduce

发表于 2018-12-19

执行相应函数，将每次的循环结果汇总并返回。

reducer 函数接收4个参数:

  • Accumulator (acc) (累计器)
  • Current Value (cur) (当前值)
  • Current Index (idx) (当前索引)
  • Source Array (src) (源数组)

求和

var sum = [0, 1, 2, 3].reduce(function (accumulator, currentValue) {
return accumulator + currentValue;
}, 0);
// 和为 6

将二维数组转变为一维数组

var flattened = [[0, 1], [2, 3], [4, 5]].reduce(
function(a, b) {
    return a.concat(b);
},
[]
);
// flattened is [0, 1, 2, 3, 4, 5]

计算数组中每个元素出现的次数

var names = ['Alice', 'Bob', 'Tiff', 'Bruce', 'Alice'];
var countedNames = names.reduce(function (allNames, name) { 
if (name in allNames) {
    allNames[name]++;
}
else {
    allNames[name] = 1;
}
return allNames;
}, {});
// countedNames is:
// { 'Alice': 2, 'Bob': 1, 'Tiff': 1, 'Bruce': 1 }

按属性对object分类

var people = [
{ name: 'Alice', age: 21 },
{ name: 'Max', age: 20 },
{ name: 'Jane', age: 20 }
];

function groupBy(objectArray, property) {
return objectArray.reduce(function (acc, obj) {
    var key = obj[property];
    if (!acc[key]) {
    acc[key] = [];
    }
    acc[key].push(obj);
    return acc;
}, {});
}

var groupedPeople = groupBy(people, 'age');
// groupedPeople is:
// { 
//   20: [
//     { name: 'Max', age: 20 }, 
//     { name: 'Jane', age: 20 }
//   ], 
//   21: [{ name: 'Alice', age: 21 }] 
// }

数组去重

let arr = [1,2,1,2,3,5,4,5,3,4,4,4,4];
let result = arr.sort().reduce((init, current)=>{
    if(init.length===0 || init[init.length-1]!==current){
        init.push(current);
    }
    return init;
}, []);
console.log(result); //[1,2,3,4,5]

按顺序运行Promise

function runPromiseInSequence(arr, input) {
return arr.reduce(
    (promiseChain, currentFunction) => promiseChain.then(currentFunction),
    Promise.resolve(input)
);
}

// promise function 1
function p1(a) {
return new Promise((resolve, reject) => {
    resolve(a * 5);
});
}

// promise function 2
function p2(a) {
return new Promise((resolve, reject) => {
    resolve(a * 2);
});
}

// function 3  - will be wrapped in a resolved promise by .then()
function f3(a) {
return a * 3;
}

// promise function 4
function p4(a) {
return new Promise((resolve, reject) => {
    resolve(a * 4);
});
}

const promiseArr = [p1, p2, f3, p4];
runPromiseInSequence(promiseArr, 10)
.then(console.log);   // 1200

export const VS default export

发表于 2018-12-19

声明式export对比default export。

　　export const 是用const关键字命名的export。那什么是default export呢，每个文件，你都能有一个default export。当你导入时，你必须明确的写出文件的名字，就像这样：import MyDefaultExport from “./MyFileWithADefaultExport”;但是你在导入的时候不局限于一种命名。

声明export（export const）

　　当你使用这种规则的时候，你能够在每个文件有多种声明的导入。当你导入明确的export时，你得这样准备。

  • 导入多重声明exports：import {MyClass, MyOtherClass} from “./MyClass”;
  • 当有多个文件声明，但是你如果只想要一个声明的时候，你可以使用as：import {MyClass2 as MyClass2Alias} from “./MyClass2”;
  • 当你想要把一个文件里面的所有的模块都导入出来时，你可以使用as，把它们映射到一个对象上：import * as MyClasses from “./MyClass”;

　　当你想导入MyClasses.MyClass and MyClass.MyOtherClass时，你能使用default export 或者声明式export，再或者在同时你可以使用这两种规则。句法规则上的优势，default export 更轻量更加的简约，这是为什么它被大量使用的原因。
　　note：default export 实际上也是声明式export，只不过是用default 声明，这也是为什么你能够这样导入模块：import {default as MyDefaultExport} from “./MyFileWithADefaultExport”;

什么是normalize

发表于 2018-12-19

　

　　normalize.css是一个轻量级css文件，它在html元素默认的样式中，提供了更好的跨浏览器一致性。它是现代化的，为html5准备的，相比传统的reset.css，normalize.css是更好的选择。详见戳：
　　Normalize.css project site
　　Normalize.css source on GitHub
　　normalize.css目前被 Twitter Bootstrap, HTML5 Boilerplate, GOV.UK, Rdio, CSS Tricks当做标准化样式，当然还有许多框架,工具，站点使用。

Overview

　　normalize.css 相比于reset.css是一个更好的选择。normalize.css是由 @necolas and @jon_neal 在不同浏览器默认样式经过了100个小时大量的研究总结出来。
normalize.css的目标有如下几点:

  • 保存了浏览器有用的样式，而不是清除。
  • 为更加多的html元素初始化了样式
  • 修复了bug和大部分浏览器的不一致性
  • normalize.css运用微妙的改进提高了css可用性
  • 解释了有评论和详细文档的代码

　　它提供支持了更多的浏览器（包括移动端浏览器）和标准化html5元素，排版，列表，内联元素(embedded content),表单和表格。
尽管normalize.css基于标准化的目的，但是它使用了更好的实用默认样式。

Normalize 对比 Reset

　　reset 使用了一种同质的样式重置了几乎所有元素的默认样式。相反，normalize保留下来了许多有用的默认样式,这就意思着你不要重新为所有的排版元素申明样式。
　　当一个元素在不同的浏览器有不同的样式时，normalize目标就是保持这些样式的一致性和符合现代浏览器的标准。
　　normalize 修复了共同的bug
　　它修复了pc端和移动端浏览器的reset所不能修复的bug。这其中包括了html5元素的显示设置，预定义文本的font-size,SVG在IE9上的溢出问题，和许多跟表单相关的跨浏览器问题和一些操作系统问题。
　　举个例子,normalize是如何处理html5新的表单类型serch的浏览器一致性。

/**
 * 1. Addresses appearance set to searchfield in S5, Chrome
 * 2. Addresses box-sizing set to border-box in S5, Chrome (include -moz to future-proof)
 */
input[type="search"] {
    -webkit-appearance: textfield; /* 1 */
    -moz-box-sizing: content-box;
    -webkit-box-sizing: content-box; /* 2 */
    box-sizing: content-box;
}
/**
 * Removes inner padding and search cancel button in S5, Chrome on OS X
 */
input[type="search"]::-webkit-search-decoration,
input[type="search"]::-webkit-search-cancel-button {
    -webkit-appearance: none;
}

Resets often fail to bring browsers to a level starting point with regards to how an element is rendered.特别是表单，normalize能提供一些有意义的援助。

normalize 不会紊乱你的调试工具

　　一个普遍的问题就是，当使用reset的时候，会在你的浏览器调试工具中产生大量的继承链。
　　上图就是使用reset样式时，调试工具状况，如果使用normalize，就不会有上述问题，因为normalize有针对的使用样式和慎用多重选择器。

normalize它是模块化的

　　这个项目已经被划分成相对独立的模块，让你很容易清楚地了解到哪些元素需要何种样式。
　　更进一步来说，它会潜移默化的去除如果在你的网站从不需要的（如,表单正常化）.

normalize 拥有详细的文档说明

　　normalize css 代码是基于详细的跨浏览器研究和科学方法的测试。文档有着非常详细的说明，并且在 GitHub Wiki 有着更进一步的扩展说明。这意味着你能够找到并了解到每一行代码的作用，这就是为什么能了解到不同浏览器的异同，并且能够更加简单的运行测试。

怎么去使用normalize.css

　　首先，从github安装或者下载normalize（download normalize.css）,有如下两种主要的使用方式：

  • 方法1：使用normalize作为你的项目（基于css项目）的初始化样式，根据设计者的要求自定义normalize的值。
  • 方法2：如果有必要的话，直接使用未更改的normalize样式，并且在默认样式表中，将normalize权重设为最高。

结束语

　　normalize在css重置样式领域和表现力是非常新颖的。它非常值得你去尝试着用它去创新和提高表现力。
　　这个项目已经在github上开源。任何人都能够提出问题和提交修改。整个项目的改动历史对每个人都是可见的，并且修改的每条内容和愿意都可以在提交信息和中找到。

相关阅读

详细信息见默认UAstyles：WHATWG suggestions for rendering HTML documents, Internet Explorer User Agent Style Sheets,and CSS2.1 User Agent Style Sheet Defaults.

通信数据转发程序：代理、隧道、网关

http通信是，除了客户端和服务器之外，还有一些用于通信数据转发的应用程序，如代理，隧道，网关。

代理：

总的一句话来说，充当”中间人”的角色.代理服务器的基本行为就是就收客户端发来的请求，然后转发给其他的服务器。并且代理不会改变请求的uri，直接将请求发送给前方持有资源的源服务器。如请求经过像锁链一样的代理时，转发时，需要附加via首部字段以标记处经过的代理服务器信息。

为什么使用代理服务器？

1。利用缓存技术减少网络带宽的流量。
2。组织内部针对特定网站的访问控制，以获取访问日志为主要目的。

缓存代理:

代理转发响应是，会预先将资源的副本保存在代理服务器上，当再次接收到相同资源的请求时，代理服务器不会从源服务器那里请求资源，而是将之前的缓存的资源作为响应返回。

touch相关

发表于 2018-12-13

H5 touch相关

touchEvent

TouchEvent 是一类描述手指在触摸平面（触摸屏、触摸板等）的状态变化的事件。这类事件用于描述一个或多个触点，使开发者可以检测触点的移动，触点的增加和减少，等等。

每 个 Touch 对象代表一个触点; 每个触点都由其位置，大小，形状，压力大小，和目标 element 描述。 TouchList 对象代表多个触点的一个列表.

属性列表

  • TouchEvent的属性继承了 UIEvent 和 Event。
    TouchEvent.altKey 只读。
  • 布尔值，指明触摸事件触发时，键盘 alt 键是否被按下。
    TouchEvent.changedTouches 只读。
  • 一个 TouchList 对象，包含了代表所有从上一次触摸事件到此次事件过程中，状态发生了改变的触点的 Touch 对象。
    TouchEvent.ctrlKey 只读。
  • 布尔值，指明触摸事件触发时，键盘 ctrl 键是否被按下。
    TouchEvent.metaKey 只读。
  • 布尔值，指明触摸事件触发时，键盘 meta 键 （Wikipedia - meta Key）是否被按下。
    TouchEvent.shiftKey 只读。
  • 布尔值，指明触摸事件触发时，键盘 shift 键是否被按下。
    TouchEvent.targetTouches 只读。
  • 一个 TouchList 对象，是包含了如下触点的 Touch 对象：触摸起始于当前事件的目标 element 上，并且仍然没有离开触摸平面的触点。
    TouchEvent.touches 只读。
  • 一 个 TouchList 对象，包含了所有当前接触触摸平面的触点的 Touch 对象，无论它们的起始于哪个 element 上，也无论它们状态是否发生了变化。

触摸事件的类型

为了区别触摸相关的状态改变，存在多种类型的触摸事件。可以通过检查触摸事件的 TouchEvent.type 属性来确定当前事件属于哪种类型

touchstart

当用户在触摸平面上放置了一个触点时触发。事件的目标 element 将是触点位置上的那个目标 element

touchend

当一个触点被用户从触摸平面上移除（当用户将一个手指离开触摸平面）时触发。当触点移出触摸平面的边界时也将触发。例如用户将手指划出屏幕边缘。
事件的目标 element 和这个 touchend 事件对应的 touchstart 事件的目标 element 相同，哪怕 touchend 事件触发时，触点已经移出了该 element 。
已经被从触摸平面上移除的触点，可以在 changedTouches 属性定义的 TouchList 中找到。

touchmove

当用户在触摸平面上移动触点时触发。事件的目标 element 和这个 touchmove 事件对应的 touchstart 事件的目标 element 相同，哪怕当 touchmove 事件触发时，触点已经移出了该 element 。
当触点的半径、旋转角度以及压力大小发生变化时，也将触发此事件。
touchcancel
当触点由于某些原因被中断时触发。有几种可能的原因如下（具体的原因根据不同的设备和浏览器有所不同）：
由于某个事件取消了触摸：例如触摸过程被一个模态的弹出框打断。
触点离开了文档窗口，而进入了浏览器的界面元素、插件或者其他外部内容区域。
当用户产生的触点个数超过了设备支持的个数，从而导致 TouchList 中最早的 Touch 对象被取消。
